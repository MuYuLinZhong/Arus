# 设计文档归档

本目录用于存放从《设计大纲》中剥离的专项设计文档，便于独立演进与版本管理。

## 目录结构

| 目录/文件 | 说明 |
|-----------|------|
| `database/` | 数据库专项设计 |
| `database/01-数据库总体设计.md` | 数据库总体设计说明书：分库分表、设备类型扩展、日志与业务分离、监控与数据流预留 |
| `web/` | 前端专项设计 |
| `web/01-前端交互设计.md` | 前端交互设计说明书：技术选型、目录结构、路由与页面、组件层次、状态与数据流、API 对应、交互模式 |
| `server/` | 服务端专项设计 |
| `server/01-服务端总体设计.md` | 服务端总体设计说明书：架构分层、中间件、认证会话、挑战-应答协议、管理后台、消息队列、安全与可观测性 |
| `server/02-Go与依赖库入门.md` | Go 语法、Gin、GORM、RabbitMQ 基本原理与在本项目中的用法（入门向） |
| `server/03-Go语法学习笔记.md` | 学习项目代码过程中遇到的 Go 语法疑问汇总：方法接收者、接口、goroutine、channel 等 17 个知识点 |

## 与主设计大纲的关系

- **设计大纲**（项目根目录 `设计大纲.md`）：整体技术设计，包含架构、接口、Session、限流、业务逻辑、前端等。
- **本目录**：对其中可独立演进的模块做细化与解耦设计，避免耦合在同一文档中不利于可持续演进。

## 使用说明

- 数据库相关决策、表结构变更、分库分表方案以 `database/` 下文档为准。
- 服务端架构、中间件链、业务逻辑、安全设计以 `server/` 下文档为准。
- 前端路由、组件、状态、API 与交互模式以 `web/` 下文档为准。
- 开发时请参考对应专项文档作为最新基线。
